# Memory Folder

This folder contains important reference documents for the Alchemy project. These documents serve as a knowledge base and should be consulted regularly to maintain consistency and follow established patterns.

## Contents

- **architect.md**: Technical architecture documentation, including project structure, component organization, and development guidelines.
- **memory.md**: Record of key decisions, requirements, and changes made to the project over time.

## Usage Guidelines

### When to Read

These documents should be read:
- Before starting work on a new feature
- When making significant changes to existing components
- When onboarding new team members
- When planning future development

### When to Update

These documents should be updated:
- When new architectural decisions are made
- When significant changes are implemented
- When new requirements are established
- When important bugs are fixed
- When refactoring or optimizing existing code

### How to Update

1. Make changes to the relevant document(s)
2. Keep the format consistent
3. Add dates to new entries when possible
4. Be concise but thorough
5. Organize information logically

## Importance

Maintaining these documents is crucial for:
1. **Knowledge Preservation**: Prevents loss of important context and decisions
2. **Consistency**: Ensures the project follows established patterns
3. **Onboarding**: Helps new team members understand the project quickly
4. **Decision Making**: Provides context for future decisions

## Rule for AI Assistants

AI assistants working on this project should:
1. Read the memory folder documents before making significant changes
2. Update the documents when implementing new features or making important changes
3. Reference these documents when explaining decisions
4. Suggest improvements to the documentation when appropriate
